🔁 v-if vs v-show dans Vue.js 3

État actuel : {{ isVisible ? 'Visible' : 'Caché' }}

📌 v-if

✅ Ce paragraphe utilise v-if. Il est visible.

❌ Ce paragraphe (v-else) apparaît quand la condition est fausse.

🔍 v-if : l'élément est ajouté ou supprimé du DOM selon la condition.

🎭 v-show

✅ Ce paragraphe utilise v-show. Il est visible.

🔍 v-show : l'élément reste toujours dans le DOM, seul son CSS display bascule entre none et la valeur d'origine.

📖 Nuances importantes entre v-if et v-show

  • v-if : rendu conditionnel réel → l'élément est détruit/recréé lors du basculement. Plus coûteux pour des changements fréquents.
  • v-show : simple bascule CSS (display: none) → l'élément reste dans le DOM. Plus performant pour des toggles répétés.
  • v-if peut être utilisé avec v-else / v-else-if, ce que v-show ne permet pas.
  • v-if n'est pas adapté si l'élément contient des données ou des composants lourds à recréer souvent.
  • Astuce : ouvrez l'inspecteur d'éléments du navigateur et cliquez sur le bouton. Observez que le bloc v-if disparaît/ réapparaît du DOM, tandis que le bloc v-show reste présent mais avec style="display: none;".
